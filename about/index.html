About Jeff Thoensen

QA engineer, context-driven testing, Playwright, API, and exploratory work

I work on QA that reduces risk instead of padding test counts. I build Playwright suites, wire them into CI, and spend a lot of time on exploratory testing because that is where real issues show up. I care about tests that people trust (flaky suites do the opposite). I check UI mocks early, I keep suites small on purpose, and I treat test code like product code (reviews, refactors, small commits).

Tools I use regularly: Playwright, GitHub Actions, Allure reports. Languages: JavaScript and TypeScript mainly, with some Python when it helps. If you want the longer version of how I approach automation, read the post below.
